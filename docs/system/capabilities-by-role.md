# 権限別に「何ができるか」（能力マトリクス）

本文書は **MVP の意図**に基づく。**API・サーバ側の判定が正**であり、UI はそれに追従する（[`security-and-auth.md`](security-and-auth.md)）。

## ロール定義（組織スコープ）

| ロール | ざっくりした責務 |
|--------|------------------|
| **owner** | 組織の最終責任（危険操作・譲渡は将来拡張として本表では「設定の最高権限」に含める） |
| **admin** | メンバー・ロール・イベント作成・トークン運用・サテライト健全性の監視 |
| **moderator** | 日常スレッド等の**モデレーション**（投稿の非表示・ロック等。MVP で範囲を入れるかは実装時に FR と同期） |
| **member** | 長命スレッドへの投稿・閲覧（組織ポリシー内）、イベント**閲覧**（公開設定に依存） |
| **参加トークン保有者** | **Google ログイン不要でもよい**書込主体。イベント **live** 中のサテライト書込（死因ログ等）の根拠は **トークンが正**（F-06） |
| **未ログイン** | 公開設定に応じた **読取中心**（`archived` 後の全世界読取の意向に整合） |

※ 実装時は Better Auth のユーザー ID と、組織未所属ユーザー、トークンのゲスト ID の扱いをスキーマで固定する。

## イベントフェーズ別の一文まとめ

| フェーズ | サテライト書込 | サテライト読取（目安） |
|----------|----------------|------------------------|
| `draft` | 運営のみ or 不可（実装で確定） | 運営 |
| `live` | **有効トークン**（＋運営ロール）。補助条件は F-06 | 公開ポリシーに従う |
| `archived` | **すべて拒否**（F-09） | **全世界／共有 URL** を志向。キャッシュは NFR |

## 実装チェックリスト（PR 時）

- [ ] 各 `POST`/`PATCH`/`DELETE` が上表と矛盾しない
- [ ] トークン検証がクライアント推測に依存していない（サーバのみ）
- [ ] `archived` の拒否がサテライト・プライマリの両面で齟齬がない（イベントメタが `archived` なのにサテライトだけ開いている等）
