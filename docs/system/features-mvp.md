# MVP 機能要件

実装時のトレーサビリティ用 ID。受け入れ条件は [`capabilities-by-role.md`](capabilities-by-role.md)・[`screens-and-ui.md`](screens-and-ui.md) と同期させる。

| 要件ID | 説明 | 備考 |
|--------|------|------|
| FR-01 | 組織（グループ）CRUD、メンバー・ロール管理 | Google ログインユーザーを組織に紐付け |
| FR-02 | 長命スレッドの作成・投稿・閲覧（**プライマリ DB**） | チャンネル複数化はスキーマ上後方互換で拡張可 |
| FR-03 | イベントの作成・`draft` / `live` / `archived` 遷移 | 手動またはスケジュールで閉じる |
| FR-04 | イベントサテライト DB のプロビジョニング | 失敗時リトライ、主催向けステータス表示 |
| FR-05 | 死因・層・ビルド系ログの投稿・集計・閲覧（**サテライト**） | MVP の「顔」 |
| FR-06 | 参加トークン発行と検証（**書込 API**） | QR・短命 JWT 想定 |
| FR-07 | `archived` 後は書込禁止、閲覧は継続 | 静的スナップショットは最適化オプション |
| FR-08 | サインイン（Better Auth × Google） | GCP Console OAuth クライアント |

### データ面との対応（要約）

| 要件群 | 主 DB |
|--------|--------|
| FR-01, FR-02, FR-03（メタ）, FR-04（制御）, FR-06（発行記録）, FR-08 | プライマリ |
| FR-05, FR-03（当日ログ）, FR-06（サテライト書込検証の到達）, FR-07（サテライト側拒否） | サテライト（イベント単位） |
