# 製品概要・スコープ

## 価値提案（一文）

**ひとつのコミュニティの下で、日常のスレッドは続き、イベントだけは会場と締切で書き込みを閉じる。Turso により記録を分離しつつ、運営の見え方は一つにまとめる。**

## 名称・コピー

| 項目 | 内容 |
|------|------|
| サービス名 | **Threadhall**（確定） |
| 意味づけ | 「Hall」＝コミュニティの広間。イベントと日常が**並立**する命名 |
| サブコピー例 | コミュニティの長く続くスレッドと、会場だけのイベントログ。 |

## 第一ペルソナ

- **イベント主催／コミュニティ運営**が第一。作者・一般参加者は副次扱い（ただし参加トークン経由の書込はプロダクトの要）。

## プロダクト二層

| 層 | 説明 |
|----|------|
| **(A)** | オフライン／コミュニティ**イベント**向けスレッド・ログ（締切・**アーカイブ**あり） |
| **(B)** | **長命の日常スレッド**（組織の継続的な会話） |

同一**組織**配下で束ねる。詳細は [`domain-model.md`](domain-model.md)・[`data-and-lifecycle.md`](data-and-lifecycle.md)。

## スコープ（MVP に含める意図）

- 組織の作成、メンバー招待、**複数ロール**（owner / admin / moderator / member 等をスキーマで許容）
- 長命スレッド（組織配下）と、イベントにひもづくスレッド／ログ用途
- イベント作成時の **サテライト DB プロビジョニング**、ダッシュボードでの見える化（失敗リトライ含む）
- 参加トークンに基づく書込ゲート、イベントフェーズに基づく拒否
- Google Cloud 上での API ホスティングと秘密情報の集約（Secret Manager）

能力の境界は [`capabilities-by-role.md`](capabilities-by-role.md)、画面単位は [`screens-and-ui.md`](screens-and-ui.md)。

## 非目標（現時点で約束しない）

- Discord／Slack の完全代替、リアルタイム全体同期の**保証**
- GeoIP のみを「法的・完全な」参加証明として売ること
- Turso へのバイナリ大容量アセットの直保存（メタ＋外部ストレージが現実解）
- モデレーション全自動、公開掲示板の無秩序運用（[`security-and-auth.md`](security-and-auth.md) の脅威メモ参照）
