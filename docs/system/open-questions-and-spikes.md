## スパイクゲート（最重要の次ステップ）

**保留中**: ルート単位で「**GET は全世界（キャッシュ方針含む）**／**書込はゲート＋ Archived 拒否**」をスパイクし、確信を得てから周辺機能を増やす。

完了基準の例（実装時に具体化）:

- `archived` での変異系が API 層で一貫拒否されること
- 参加トークン無しの書込がサテライトに届かないこと
- 監査・ログにプロビジョニング失敗が残ること（運用への通知は別途）

対話により **Threadhall** 命名確定、**プライマリ＋サテライト** データ面確定、**死因ログ** を MVP 縦スライスに設定、**参加トークン優先**、**Better Auth × Google × GCP** をインフラ方針として採用。
